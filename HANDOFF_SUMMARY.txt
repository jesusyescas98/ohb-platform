4. JWT AUTHENTICATION SYSTEM
   File: src/lib/auth/jwt.ts (400 lines)

   Token Generation:
   - signAccessToken(payload, secret, expiresIn) - 15 minute tokens
   - signRefreshToken(userId, secret, expiresIn) - 7 day tokens
   - createTokenSet(payload, secret) - Both tokens for login

   Token Verification:
   - verifyToken(token, secret) - Verify and decode
   - decodeTokenUnsafe(token) - Decode without verification (debug)

   Token Utilities:
   - isTokenExpired(token)
   - shouldRefreshToken(token) - Checks 5-min threshold
   - getTokenExpirationDate(token)
   - getTokenTimeRemaining(token)

   Token Refresh:
   - refreshAccessToken(refreshToken, secret, getUserPayload)

   Features:
   - HMAC-SHA256 signing (standard JWT)
   - Compatible with both Node.js and browsers
   - Configurable token lifetimes
   - 5-minute auto-refresh threshold
   - Proper JWT header/payload/signature structure

TECHNICAL SPECIFICATIONS
================================================================================

DATABASE DESIGN
- 12 tables normalized to 3NF
- UUID primary keys for distributed systems
- Timestamps on all tables (created_at, updated_at)
- Soft-delete support (deleted_at columns)
- Status ENUM constraints for data integrity
- 20+ indexes optimized for common queries
- 3 materialized views for analytics
- Foreign key constraints with CASCADE behavior

AUTHENTICATION STRATEGY
- JWT-based (not session-based cookies)
- Access tokens: 15 minutes (fast, short-lived)
- Refresh tokens: 7 days (long-lived)
- HMAC-SHA256 cryptographic signing
- JWT verified server-side on every request
- Auto-refresh mechanism at 5-minute threshold
- Logout support via token blacklisting (future)

AUTHORIZATION MODEL
- Role-based access control (RBAC)
  - admin: Full system access
  - asesor: Create/manage own properties, assigned leads
  - cliente: View published properties, manage own enrollments
- Resource ownership-based access
  - Users can only modify their own data
  - Asesores assigned specific leads
- RLS policies enforced at database level
  - Service role key for admin operations only
- Public access for published content
  - Properties (status='activa' only)
  - Courses (published=true only)

SUPPORT & TROUBLESHOOTING
================================================================================

For SQL Schema Issues:
1. Check Supabase SQL syntax
2. Verify column names are correct
3. Check data types match examples
4. Review error message carefully
5. Try running query sections individually

For RLS Policy Problems:
1. Test policy in Supabase SQL console
2. Verify JWT claims in token
3. Check role matches policy condition
4. Review RLS_POLICIES.md examples
5. Use Supabase logs for debugging

For Query Helper Errors:
1. Verify table/column names
2. Check RLS isn't blocking
3. Review error from Supabase
4. Check filter operators are valid
5. Try query as admin (service role)

For JWT Issues:
1. Verify JWT_SECRET is set
2. Check token format (3 parts with dots)
3. Verify token isn't expired
4. Use decodeTokenUnsafe() for debugging
5. Check auth header format

For API Route Issues:
1. Check endpoint path is correct
2. Verify HTTP method (GET, POST, etc.)
3. Check Authorization header
4. Verify request body format
5. Review API_ENDPOINTS.md for examples
